Cross-platform

.NET

NEntityDb is a lightweight, extensible, and cross-platform data access technology built on the .NET platform. You can create apps for Windows, macOS, and Linux using multiple programming languages.

.NET is a free, cross-platform, open source developer platform for building many different types of applications. With .NET, you can use multiple languages, editors, and libraries to build for web, mobile, desktop, gaming, and IoT.

Languages

You can be use the C#, Visual Basic, and F# languages to write applications and libraries with NEntityDb. These languages can be used in your favorite text editor or Integrated Development Environment (IDE).

  • C# is a simple, modern, object-oriented, and type-safe programming language.
  • F# is a cross-platform, open-source, functional programming language for .NET. It also includes object-oriented and imperative programming.
  • Visual Basic is an approachable language with a simple syntax for building type-safe, object-oriented apps.

Editors

You can use the tools of Visual Studio family products for building .NET apps on any operating system. There are also command line tools and extensions for many popular editors. If you prefer to use a different editor, there are .NET command line tools and plugins for many popular editors.

The Visual Studio Marketplace has thousands of editor extensions from Microsoft and others.

Community

To extend functionality, Microsoft and others maintain a healthy package ecosystem built on .NET Standard. NuGet is a package manager built specifically for .NET that contains over 90,000 packages.

Because .NET is open source, you can join the 60,000 developers and 3,700 companies already contributing to the .NET platform.

Database abstraction layer

NEntityDb provides you with a database abstraction layer, an application programming interface which unifies the communication between a computer application and databases. NEntityDb reduce the amount of work by providing a consistent API to the developer and hide the database specifics behind this interface as much as possible.

If an application is built with NEntityDb, it is database-agnostic. If you don't use NEntityDb and you had written your database queries to work with MySQL and 2 years down the line, your database changes to Oracle, you will need to rewrite most of your database queries.

Object Relational Mapping

NEntityDb is a modern object relational mapper for .NET that enables developers to work with a database using .NET objects, and eliminating the need for most of the data-access code they usually need to write.

Object-Relational Mapping is a technique that lets you query and manipulates data from a database using an object-oriented paradigm. Most software projects need to talk to a database at some point – they usually do get queries for data, update a field or create a record.

You can use NEntityDb to persist model objects to a relational database and retrieve them, and the NEntityDb will take care of converting the data between the two otherwise incompatible states.

NEntityDb supports many database engines, see Database providers for details.

Productivity

There are a number of productivity benefits to using NEntityDb for development of databased applications.

Code reduction

The data access code is usually a significant portion of a typical application, and the time needed to write that code can be a significant portion of the overall development schedule. When using NEntityDb, the amount of code is unlikely to be reduced-in fact, it might even go up-but the NEntityDb tool generates 100% of the data access code automatically based on the data model you define, in more moments.

The following code show how to make the same query with ADO.NET and NEntityDb.

C#
//Query with ADO.NET
using (DbConnection connection = dbProviderFactory.CreateConnection("your connection string"))
{
    connection.Open();
    using (DbCommand command = connection.CreateCommand()) 
    {
        command.CommandText = "SELECT CUS_Id, CUS_FirstName, CUS_LastName, CUS_TaxCode, CUS_Email " + 
                              "FROM Customers " +
                              "WHERE CUS_FirstName LIKE '%a%' OR CUS_LastName LIKE '%a%'";
        
        List<Customer> customers = new List<Customer>();

        using (DbDataReader reader = command.ExecuteDataReader()) 
        {
            while (reader.Read()) 
            {
                Customer customer = new Customer();
                customer.Id = (int)reader["CUS_Id"];
                customer.FirstName = (string)reader["CUS_FirstName"];
                customer.LastName = (string)reader["CUS_LastName"];
                customer.TaxCode = (string)reader["CUS_TaxCode"];
                customer.Email = (string)reader["CUS_Email"];

                customers.Add(customers);
            }
        }
    }
}

//Same query with NEntityDb
using (DbInstance dbInstance = new DbInstance())
{
    List<Customer> customers = dbInstance.QueryOf<Customer>()
        .Select(c => Property.ToArray(c.Id, c.FirstName, c.LastName, c.TaxCode, c.Email))
        .Where(c => c.FirstName.Contains("a") || c.Email.Contains("a"))        
        .Page(1, 50)
        .ToList();
}

Application design

With NEntityDb, software architects will implement effective design patterns that almost force you to use good programming practices in an application. This can help support a clean separation of concerns and independent development that allows parallel, simultaneous development of application layers.

NEntityDb facilitates implementing the Domain Model pattern. Using this pattern means that you model entities based on real business concepts rather than based on your database structure. NEntityDb provide this functionality through mapping between the logical business model and the physical storage model.

Application maintainability

As software developers, we all know that we need to try and avoid duplicate code. Duplicated code is harder to update, maintain, and reuse. With NEntityDb the code easier to update, maintain, and reuse as the developer can think of, and manipulate data as objects.

All of the code generated by the NEntityDb is well-tested, so you usually don’t need to worry about testing it extensively.

Portability

Other main benefit of using NEntityDb is portability, because it makes switching to another database possible, even in the middle of a project. Suppose that you need to write a quick prototype for an application, but the client hasn't decided yet which database system would best suit his needs. You can start building your application with SQLite, for instance, and switch to MySQL, PostgreSQL, or Oracle when the client is ready to decide.

.NET LINQ Expressions

NEntityDb uses .NET LINQ Expressions to query and save data from database. .NET LINQ Expressions allows you to use C# (or your .NET language to choice) to write strongly typed queries based on your entities. A representation of the .NET Expression query is passed to the database provider, to be translated to SQL language.

The .NET LINQ Expressions enables language-level code expressions to be represented as objects in the form of expression trees. Because a .NET LINQ Expression is declarative, it usually allows you to write concise code that is easy to understand and maintain.

.NET LINQ Expressions are strongly typed and expedites development time by catching errors at compile time and includes IntelliSense & Debugging support. Strongly typed expressions ensure access to values as the correct type at compile time & prevents type mismatch errors being caught when the code is compiled rather at run-time.

The following code show how to update data using .NET LINQ Expressions with NEntityDb.

C#
using (DbInstance dbInstance = new DbInstance())
{    
    int rowsAffected = dbInstance.NonQueryOf<Customer>()
        .Set(() => new Customer
        {
            Email = "cguiet@gov.uk",
            TaxCode = "3720045188"
        })
        .Where(c => c.Id == 10)
        .Update()
        .RowsAffected;
}

Fluent API

With NEntityDb you can use Fluent APIs both to configure your model and to query and save data. Using Fluent APIs in NEntityDb makes code readability easier.

A fluent interface is a method of designing object-oriented APIs based extensively on method chaining with the goal of making the readability of the source code to that of ordinary written prose, essentially creating a domain-specific language within the interface.

Three reasons why we use Fluent APIs in NEntityDb:

  • Readability is a primary design concern
  • Fluent APIs emphasize the purpose of your code
  • Fluent APIs let you wield complex concepts with ease

The following code shows how to query data using Fluent API.

C#
using (DbInstance dbInstance = new DbInstance())
{    
    List<Customer> customers = dbInstance.QueryOf<Customer>()
        .Select(c => Property.ToArray(c.Id, c.FullName, c.Age, c.BirthDate, c.TaxCode, c.Email))
        .Where(c => c.Age > 18 && c.FullName.Contains("a"))
        .OrderBy(c => c.FullName)
        .OrderByDescending(c => c.Age)
        .Page(1, 50)
        .ToList();
}

The following code shows how to configure a model using Fluent API.

C#
public class DbCustomer: DbEntity<Customer>
{
    public DbCustomer()
    {
        this.ToTable("Customers");

        this.Property(c => c.Id).ToColumn("CUS_Id").IsIdentity();
        this.Property(c => c.FirstName).ToColumn("CUS_FirstName");
        this.Property(c => c.LastName).ToColumn("CUS_LastName");
        this.Property(c => c.TaxCode).ToColumn("CUS_TaxCode");
        this.Property(c => c.Email).ToColumn("CUS_Email");
        this.Property(c => c.IsActive).ToColumn("CUS_IsActive");
        this.Property(c => c.BirthDate).ToColumn("CUS_BirthDate");
        this.Property(c => c.DeathDate).ToColumn("CUS_DeathDate");
        this.Property(c => c.Points).ToColumn("CUS_Points");
        this.Property(c => c.CreationDate).ToColumn("CUS_CreationDate");
        
        this.WithPrimaryKey(c => c.Id);

        this.WithForeignKey<Address>(c => c.MainAddress)
            .Is<Customer, Address>((c, a) => c.Id == a.Customer.Id && a.IsMain.Value);

        this.Include<Address>(a => a.Customer, c => c.Addresses);
    }
}

Static and dynamic queries

NEntityDb allows you to query or save data both statically and dynamically.

Static queries

NEntityDb uses .NET LINQ Expressions to define static queries. Static queries with .NET LINQ Expressions prevents type mismatch errors being caught when the code is compiled rather at run-time and includes IntelliSense & Debugging support.

The following code shows how to define and run a static query.

C#
//Static
using (DbInstance dbInstance = new DbInstance())
{
    List<Customer> customers = dbInstance.QueryOf<Customer>()
        .Select(c => Property.ToArray(c.Id, c.FirstName, c.LastName, c.TaxCode, c.Email))
        .Where(c => c.FirstName.Contains("a") || c.Email.Contains("a"))
        .OrderByDescending(c => c.Age)
        .OrderBy(c => c.FirstName)
        .OrderByDescending(c => c.Id)
        .Page(1, 50)
        .ToList();
}

Dynamic queries

If you need it, with NEntityDb, you can also define and run dynamic queries. In NEntityDb any statically defined query can be defined dynamically, and vice versa. You can also add dynamically parts to the definition of a static query.

The following code shows how to define and run the a dynamic query.

C#
//Dynamic
using (DbInstance dbInstance = new DbInstance())
{
    QueryRequest<Customer> request = dbInstance.QueryOf<Customer>();
    request.Properties = Property.GetProperties("Id", "FirstName", "LastName", "TaxCode", "Email");
    request.Filter = new List<Filter>
    {
        new ValueFilter("FirstName", FilterOperator.Contains, "a"),
        new ValueFilter("Email", FilterOperator.Contains, "a", LogicOperator.Or),
    };
    request.Filter.Enclose();
    request.Order = new List<Order>
    {
        new Order("Age", OrderType.Descending),
        new Order("FirstName"),
        new Order("Id", OrderType.Descending)
    };
    request.PageInfo = PageInfo.FromPage(1, 50);

    List<Customer> customers = request.ToList();
}
